Installation der Software

Die Software wurde auf einem bereiterstellten VM, die auf Ubuntu „Focal Fossa 20.04 lts“ basiert ist, installiert.

Vor der Installation des aktuellen Pakets ist es wichtig, das Basissystem mit dem Befehl in Kommandozeile „sudo apt-get update && sudo apt-get upgrade -y“ zu aktualisieren.

Elasticsearch Installation

$ apt install apt-transport-https sudo wget
$ echo "deb https://artifacts.elastic.co/packages/7.x/apt stable main" | tee -a /etc/apt/sources.list.d/elastic-7.x.list
$ wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | apt-key add
$ apt update
$ apt install elasticsearch
$ /usr/share/elasticsearch/bin/elasticsearch-plugin install ingest-attachment
$ systemctl start elasticsearch
$ systemctl enable elasticsearch

Zammad Installation

$ apt update
$ apt install zammad

Zammad anbinden 

# Set the Elasticsearch server address
$ zammad run rails r "Setting.set('es_url', 'http://localhost:9200')"

# Build the search index
$ zammad run rake searchindex:rebuild

Optionale Einstellungen

# HTTP Basic
$ zammad run rails r "Setting.set('es_user', '<username>')"
$ zammad run rails r "Setting.set('es_password', '<password>')"

Zammad Dienste verwalten

$ # Zammad service to start all services at once
$ systemctl (status|start|stop|restart) zammad

$ # Zammads internal railsserver
$ systemctl (status|start|stop|restart) zammad-web

$ # Zammads scheduler - relevant for all delayed- and background jobs
$ systemctl (status|start|stop|restart) zammad-worker

$ # Zammads websocket server for session related information
$ systemctl (status|start|stop|restart) zammad-websocket

Einloggen bei Zammad

Vor der Anmeldung der Users wurde die nachfolgende Konfiguration in der Google Workspace angepasst:

  •          Einrichtung der SSO Gruppe für Zammad

  •          Hinzufügen der Finanzmitarbeiter in dieser Gruppe

  •          Freischaltung der SSO Gruppe für SAML

Mit der folgenden URL https://accounting.socialchain.de/#login in einem

beliebigen Webbrowser können die Users sich bei Zammad anmelden.

Dazu sind die nachgeschriebenen Steps zu folgen:

Die Users sollen nur auf das rotmarkierte Zeichen klicken und sich mit

deren „Google Workspace“ Accounts anmelden.

 

 

Nach dem erfolgreichen Einloggen sieht man diese Fenster, womit man offiziell mit Zammad arbeiten kann.

Groups, Triggers und Roles einrichten

Laut der Anforderung des Finanzteams wurden die Gruppen aus der gesendeten Excelliste „Companies.xlsx“ von Marta, konfiguriert. Referenzen siehe den Abschnitt „Anhänge“ in dieser Dokumentation!

Durch die Einrichtung der Rollen können spezifische Berechtigungen definiert werden, in der eine oder mehrere Gruppen für bestimmte Mitarbeiter freigeschaltet werden, damit können sie die Rechnungen/Anfragen, die an ihre Gruppen zugewiesen sind, empfängt, und bearbeitet werden.  

Für die automatische Zuordnung der Mails wurden „Triggers“ automatische Auslöser in Zammad konfiguriert.  

Mit Hilfe der Konfiguration der automatischen Auslöser wurden Tickets an gezielten Gruppen weitergleitet, was wirklich ein super Funktion des Ticketsystems ist!

In dem nachfolgenden illustrierten Beispiel werden alle Tickets, die an accounting@lumaland.de verschickt werden, der Gruppe „Accounting Social Chain“ und den Mitarbeitern, die für die diese Gruppe freigeschaltet worden sind, zugeordnet.
